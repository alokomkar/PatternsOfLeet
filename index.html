1. The Two Pointers Pattern

🧠 Secret Clues to Spot This Pattern:

• Data layout is completely linear (Array, String, Linked List).
• Inputs are explicitly Sorted or sorting provides instant optimization.
• Target requires inspecting matching ends concurrently to find a pair or threshold.

Two Sum II - Input Array Is Sorted EASY

"Given a 1-indexed sorted array, find two numbers that add up to a target."

Pattern Hack: The sorted guarantee allows converging endpoints directly without allocating any extra storage space.

fun twoSum(numbers: IntArray, target: Int): IntArray {
    var left = 0
    var right = numbers.size - 1

    while (left < right) {
        val currentSum = numbers[left] + numbers[right]
        if (currentSum == target) {
            return intArrayOf(left + 1, right + 1)
        } else if (currentSum < target) {
            left++
        } else {
            right--
        }
    }
    return intArrayOf()
}
Container With Most Water MEDIUM

"Find two lines that form a container storing the maximum amount of water."

Pattern Hack: Maximize width by starting at the absolute margins, then progressively drop the smaller wall constraint.

fun maxArea(height: IntArray): Int {
    var left = 0
    var right = height.size - 1
    var maxWater = 0

    while (left < right) {
        val width = right - left
        val currentHeight = minOf(height[left], height[right])
        maxWater = maxOf(maxWater, width * currentHeight)

        if (height[left] < height[right]) {
            left++
        } else {
            right--
        }
    }
    return maxWater
}

2. Sliding Window

🧠 Secret Clues to Spot This Pattern:

• Demands sequential subsegments ("substring" or "subarray").
• Looking for dynamic optimization expressions like "longest" or "shortest" subsets.

Maximum Average Subarray I EASY

"Find a contiguous subarray of fixed length K with the maximum average value."

Pattern Hack: Keeps frame size exactly at K elements. Drops trailing values dynamically as incoming updates arrive.

fun findMaxAverage(nums: IntArray, k: Int): Double {
    var runningSum = 0.0
    for (i in 0 until k) runningSum += nums[i]
    var maxSum = runningSum

    for (i in k until nums.size) {
        runningSum += nums[i] - nums[i - k]
        maxSum = maxOf(maxSum, runningSum)
    }
    return maxSum / k
}
Longest Substring Without Repeating Characters MEDIUM

"Find the length of the longest substring without repeating characters."

Pattern Hack: Slide right to grow scope, and shrink left boundaries immediately whenever duplicates break unique map rules.

fun lengthOfLongestSubstring(s: String): Int {
    var left = 0
    var maxLength = 0
    val uniqueChars = HashSet()

    for (right in s.indices) {
        val incomingChar = s[right]
        while (uniqueChars.contains(incomingChar)) {
            uniqueChars.remove(s[left])
            left++
        }
        uniqueChars.add(incomingChar)
        maxLength = maxOf(maxLength, right - left + 1)
    }
    return maxLength
}

3. Prefix Sum

🧠 Secret Clues to Spot This Pattern:

• Needs multiple arbitrary slice calculations over stationary targets.
• Handles sequence tracking parameters containing negative intervals where standard window conditions break.

Range Sum Query - Immutable EASY

"Calculate sum of elements of nums between indices left and right inclusive."

Pattern Hack: Converts linear iteration costs down to single subtraction queries by storing running total counts.

class NumArray(nums: IntArray) {
    private val prefix = IntArray(nums.size + 1)

    init {
        for (i in nums.indices) {
            prefix[i + 1] = prefix[i] + nums[i]
        }
    }

    fun sumRange(left: Int, right: Int): Int {
        return prefix[right + 1] - prefix[left]
    }
}
Subarray Sum Equals K MEDIUM

"Return total number of continuous subarrays whose sum equals K."

Pattern Hack: Combines a historical tracking Map with rolling prefix valuations to look backwards for target segments.

fun subarraySum(nums: IntArray, k: Int): Int {
    var count = 0
    var currentSum = 0
    val prefixMap = HashMap()
    prefixMap[0] = 1

    for (num in nums) {
        currentSum += num
        if (prefixMap.containsKey(currentSum - k)) {
            count += prefixMap[currentSum - k]!!
        }
        prefixMap[currentSum] = prefixMap.getOrDefault(currentSum, 0) + 1
    }
    return count
}

4. Fast & Slow Pointers

🧠 Secret Clues to Spot This Pattern:

• Involves linked sequences or arrays whose elements reference other index locations directly.
• Tasks require finding midpoints or identifying cycles/loops using O(1) Auxiliary Space.

⚠️ Google-Tier Edge Case Warning:

This array cycle algorithm relies on a strict mathematical constraint: the input array must contain at least one duplicate. If you pass an array with completely unique elements, the pointers will run in a straight line and crash with an IndexOutOfBoundsException once they run off the end.

Linked List Cycle EASY

"Determine if a linked list contains a loop cycle structure."

Pattern Hack: Fast pointer takes 2 steps while slow takes 1 step. If a loop exists, they are mathematically guaranteed to crash into each other.

fun hasCycle(head: ListNode?): Boolean {
    var slow = head
    var fast = head

    while (fast != null && fast.next != null) {
        slow = slow?.next
        fast = fast.next?.next
        if (slow == fast) return true
    }
    return false
}
Find the Duplicate Number MEDIUM

"Find the single repeated number in an array of size n+1 without modifying it and using O(1) space."

Pattern Hack: Map values to node targets (index ➔ nums[index]). The duplicate number naturally serves as the entrance to an infinite loop trap.

fun findDuplicate(nums: IntArray): Int {
    var slow = nums[0]
    var fast = nums[0]

    // Phase 1: Establish intersection collision point
    do {
        slow = nums[slow]
        fast = nums[nums[fast]]
    } while (slow != fast)

    // Phase 2: Find loop entrance point
    var pointer1 = nums[0]
    var pointer2 = slow
    while (pointer1 != pointer2) {
        pointer1 = nums[pointer1]
        pointer2 = nums[pointer2]
    }
    return pointer1
}
📊 Structural Execution Dry-Run

Input: nums = [2, 5, 1, 4, 3, 2]

Graph Traversal Path: Index 0 ➔ Node(2) ➔ Node(1) ➔ Node(5) ➔ Node(2) [Trapped in Cycle!] Visual Graph Layout: (Start) 0 ──> 2 ──> 1 ──> 5 ▲ │ └───────────┘

Phase 1 Tracking (Collision):
• Round 1: slow = nums[2] (1) | fast = nums[nums[2]] ➔ nums[1] (5)
• Round 2: slow = nums[1] (5) | fast = nums[nums[5]] ➔ nums[2] (1)
• Round 3: slow = nums[5] (2) | fast = nums[nums[1]] ➔ nums[5] (2) ➔ Match!

Phase 2 Tracking (Entrance):
Reset pointer1 to nums[0] (2), move both at speed 1. They match immediately on value 2 without jumping, pinpointing the duplicate entry point.

5. Monotonic Stack

🧠 Secret Clues to Spot This Pattern:

• Seeking the relative "next greater element" or "previous smaller element" coordinates.
• Needs to trace historical information while keeping a strict, sorted structural hierarchy layout.

Next Greater Element I EASY

"Find the first greater element to the right for values mapped inside a subset array."

Pattern Hack: Elements wait inside a decreasing stack. When a larger number arrives, it forcefully pops all smaller elements and logs them as their next greater item.

import java.util.Stack

fun nextGreaterElement(queryArray: IntArray, searchArray: IntArray): IntArray {
    val answerMap = HashMap() // [Value -> Its Next Greater Element]
    val waitingRoom = Stack()

    // Find answers for elements in the searchArray
    for (currentNum in searchArray) {
        while (waitingRoom.isNotEmpty() && waitingRoom.peek() < currentNum) {
            val rescuedNum = waitingRoom.pop()
            answerMap[rescuedNum] = currentNum
        }
        waitingRoom.push(currentNum)
    }

    // Extract answers for elements in the queryArray
    val finalResult = IntArray(queryArray.size)
    for (i in queryArray.indices) {
        finalResult[i] = answerMap.getOrDefault(queryArray[i], -1)
    }
    return finalResult
}
📊 Waiting-Room Stack Trace

Inputs: searchArray = [2, 1, 3, 4], queryArray = [1, 3]

• Step 1 (currentNum=2): waitingRoom is empty. Push 2. ➔ Stack: [2] • Step 2 (currentNum=1): 1 < 2. No pop. Push 1. ➔ Stack: [2, 1] • Step 3 (currentNum=3): 3 > 1 ➔ answerMap[1]=3. Pop 1. 3 > 2 ➔ answerMap[2]=3. Pop 2. Push 3. ➔ Stack: [3] | Map: {1➔3, 2➔3} • Step 4 (currentNum=4): 4 > 3 ➔ answerMap[3]=4. Pop 3. Push 4. ➔ Stack: [4] | Map: {1➔3, 2➔3, 3➔4}

Finally, looping over queryArray = [1, 3] extracts results `[3, 4]` via instant map lookups.

Daily Temperatures MEDIUM

"Calculate the day distance required to experience a warmer temperature reading."

Pattern Hack: Store indices instead of temperature integers directly. This allows computing linear distances ($i - \text{oldIndex}$) upon item eviction.

import java.util.Stack

fun dailyTemperatures(temperatures: IntArray): IntArray {
    val result = IntArray(temperatures.size)
    val stack = Stack() // Stores indices

    for (i in temperatures.indices) {
        while (stack.isNotEmpty() && temperatures[stack.peek()] < temperatures[i]) {
            val historicalIndex = stack.pop()
            result[historicalIndex] = i - historicalIndex
        }
        stack.push(i)
    }
    return result
}
